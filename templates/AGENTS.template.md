## Best AGENTS.md Structure

The best AGENTS.md structure is flexible and tailored to your project, but typically includes these key sections:

1. Project Overview - brief description of the codebase and architecture.
2. Development Environment - setup instructions, package managers, and workspace commands.
3. Build & Test Commands - explicit commands to run builds, tests, and validation checks.
4. Code Style & Conventions - naming conventions, formatting rules, and architectural patterns.
5. Testing Guidelines - how to write tests, where test files go, and testing standards.
6. Security Considerations - sensitive areas, security best practices, and common vulnerabilities to avoid.
7. PR/Commit Guidelines - commit message format, branch naming, and review requirements.

Remember: AGENTS.md has no required fields or strict format. Use markdown headings that make sense for your project. Keep instructions clear and actionable. Update it as your project evolves. The goal is to give agents the context they need to work effectively on your codebase.
